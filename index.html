What is EzGitDoc?

EzGitDoc is a project made with JavaScript and HTML to improve Git documentation experience. Project's target was to create page where user can prepare doc template that later can be generated and pasted into *.md file. This gives ability to prepare nice, thought schemes that will increase quality of repository for newcomers and employers checking portfolios. You don't need to install anything as the project is setup and ready to use on github-pages. It's worth mentioning that some of the output elements are using HTML tags and some not (e.g. unordered list) but that is totally fine. By making linebreak these two ways of designing document are not causing conflicts between themselves. This gives ability of mixing both approaches which sometimes I find useful. EzGitDoc in its current stage allows you to create every desired element but some of them e.g. table can't be edited inside. But still.. This leaves you opportunity to make prototype of your document which later only needs to be edited with data reducing the risk of problems. EzGitDoc code translation outputs markdown *.md files accept. EzGitDoc is using SortableJS to let you switch elements positions. For more details please see repository page.

Header Settings

Notes
  • If style is not picked, header will be built using h2 tag.

Link Settings

Notes
  • Specify the name of the link by text input.
  • Both fields are required.

Image Settings

Notes
  • If you don't have URL yet, type blank.
  • If position is not picked image will be center aligned.
  • Set only one of the axis if you want proportion to be preserved.
  • At least one axis needs to be set (it's value must be different than 0).

Table Settings

Notes
  • Table data can be filled in manually just after generating github README code.
  • Only positive integer values are accepted (different than 0).
  • In EzGitDoc v1 all cells are centered.

Code Settings

Notes
  • Language field is optional.
  • Code area must contain at least one character.
  • Different language colors are not visible on EzGitDoc v1.

Text Settings

Notes
  • Leave text field empty if you want to use lorem ipsum template.

List Settings

  • GitHub Icon and bold text
  • Point
    • Choose how each element of list should be introduced.

Notes
  • Point separator
    Jagged elements of the list must be added manually later.
  • GitHub Icon separator
    Icon can be added within generated template later by replacing Id(value put between : chars) using Ctrl+F.
  • Size of list must be integer value different than 0.
