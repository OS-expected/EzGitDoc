Definition

In short, EzGitDoc is quick and easy tool allowing to setup and generate interesting template of repository documentation. One of it's crucial advantages is that it focuses more on clicking than writing. Includes extras like fancy gallery based on the table. Plan your doc', generate code and fill the data whenever you want! Learn more about the project and other pros of using it at GitHub by clicking here

Get Started

If you are not familiar with the EzGitDoc tool, you can start by simply trying each element that can be created or take a look at Help Quick Start for very fast briefing, and optionally, visit also Examples section for short capabilities demonstration.

What is EzGitDoc?

EzGitDoc is coded with JavaScript, CSS and HTML. Project's target was to to improve Git documentation experience by developing page, where user can prepare README template, that furtherly can be generated and pasted into md file. This gives ability to prepare nice, thought schemes that will increase quality of repository for people finding solutions and employers verifying repositories. You don't need to install anything as the project is setup and ready to use on github-pages. It's worth mentioning that some of the output elements are using HTML tags and some not (e.g. unordered list) but that is totally fine. By making linebreak, these two ways of designing document are not causing conflicts between themselves(sometimes it's not obligatory, but still welcome for readability). EzGitDoc in its current stage allows you to create crucial elements. In the newest version, every element's data can be modified after rendering it on page(except image type table). EzGitDoc leaves you opportunity to make prototype of your document, which later only needs to be polished, reducing the risk of problems and necessity to look after markdown cheatsheet. EzGitDoc code translation outputs markdown that *.md files accept. To read more about advantages of using EzGitDoc, please refer to the repository page by clicking on the button below.

Quick start



EzGitDoc interface conception got fully redesigned as the sidebar(instead of navbar and top menu). First section, named Tools, trigger side modals that allow to configure elements before render. Modals can be opened/closed by clicking on the mentioned buttons or by using shortkeys combination(if you want to see hints which button triggers specific tool, it will be mentioned later). Only one modal can be opened at a time. If you make at least one element, in the right bottom corner you will find button that on click will generate the scheme output. Example:





Then you only need to click button, paste the output into doc' file and polish if necessary. Or you can choose option. and use it when needed.





Let's get back to the sidebar. Notice red button placed below logo. This option lets you clear your currently made scheme and start from the beginning. It needs confirmation, so don't worry if you click on it by mistake.

At the bottom of the sidebar there are actions that may be useful for you. From the left: allows you to close sidebar, scroll to the top or to the bottom of template and finally, additional settings. If you toggle last option, you can manage some extra options. Take a look at image below for more details or see it by yourself.





If you don't fullfil certain element pre-conditions(e.g. code tag text requirement), you will see notification in the top left corner and element won't be created until the cause is fixed. Example:





In the newest version, every element data(except image table) can be edited. To do this, click on the button placed on the left side of chosen element(marked on the image below with red ellipsis). After click, you will see modal popup where you can make changes during template creation(or you can replace data later, after pasting code into doc).





Don't forget that you can switch created elements, thanks to SortableJS library! You only need to click with mouse button element and drag by holding it, until reaching desired position. By the way.. If you don't like element you've made, you can delete it by using the red cross placed on the right side of element. Be sure to pick the correct one!





Keep in mind that you can access EzGitDoc documentation from different places. They have been marked on the picture below.










You are ready now to use EzGitDoc!

Support

Browser

It is recommended to use modern browser in order to get maximum functionality out of the project. EzGitDoc uses for example classList(HTML DOM property) which is not working on every browser version. According to w3schools documentation you must have at least Chrome 8, IE10, Firefox 3.6, Safari 5.1 or Opera 11.5 in order to properly use all implemented EzGitDoc functions.

Device

EzGitDoc user interface(UI) is built using Bootstrap and MDBootstrap frameworks that allow to make the page react very nice on different screens(act responsively). However, the project was created with purpose to be used on for e.g. PC, notebooks. The most effective(quick and comfortable) way to use EzGitDoc will be through device with bigger screen than e.g. smartphone.

Possible questions

Can I contribute?

If you would like to take part in making EzGitDoc better, feel free to mail me what would you like to change/add (in short) or that you want to help improve EzGitDoc and we will see what we can do. I am looking forward to collaboration! (probably noone will be interested but #believe)

Can I share my template?

Yes! It's part of contribution. Email me if you have made template(or create PR but please, check first exampleTemplate and emptyTemplate files to see how to prepare it properly) that you think fits best for e.g. typical open source library or web api project. People can later see the suggestions and decide whether they want predefined, verified template or custom, they own one.
