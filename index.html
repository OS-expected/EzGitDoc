Image Settings

Notes
  • If you don't have URL yet, leave blank.
  • If alignment is not picked, image will be center aligned.
  • Set only one of the axis if you want proportion to be preserved.
  • At least one axis needs to be set (it's value must be different than 0).

Link Settings

Notes
  • Specify the name of the link by text input.
  • Both fields are required.
  • You can also use TEXT element to create links with anchor tag.

Table Settings

Notes
  • Table data can be filled in manually just after generating code.

Text Settings

Notes
  • Leave text field empty if you want to use lorem ipsum template.
  • You can use HTML markup in the text area like:
    <a href="address"> text </a>   (to create link)
    <strong> text </strong>   (to bold text)
    <em> text </em>   (to emphasis text)
    <del> text </del>   (to strike text)
    etc.

What is EzGitDoc?

EzGitDoc is made with JavaScript and HTML to improve Git documentation experience. Project's target was to create page where user can prepare doc' template that can be generated, pasted into *.md file and edited later. This gives ability to prepare nice, thought schemes that will increase quality of repository for newcomers and employers checking portfolios. You don't need to install anything as the project is setup and ready to use on github-pages. It's worth mentioning that some of the output elements are using HTML tags and some not (e.g. unordered list) but that is totally fine. By making linebreak, these two ways of designing document are not causing conflicts between themselves(sometimes it's not obligatory but still welcome for readability). EzGitDoc in its current stage allows you to create almost every desired element(no horizontal rule but I don't think it's that necessary). Some of them e.g. table can't be edited inside(for now). But still.. This leaves you opportunity to make prototype of your document which later only needs to be polished with data, reducing the risk of problems and necessity to look after markdown cheatsheet. EzGitDoc code translation outputs markdown that *.md files accept. To read more about advantages of using EzGitDoc, please see repository page by clicking on the button below.

Quick start



In the example above, buttons placed between red and cyan are the one responsible for creating desired elements of the template. In each white tile, there is a note about key combinations that can be used in order to trigger specific modal. Only one modal can be opened at a time, so feel free to use combinations to show other side windows and close one that's actually opened. Using the same combination that triggered modal will work as close function. If you make at least one element, in the right bottom corner you will see button that on click will generate the scheme output. Example:





Then you only need to click button, paste the output into doc' file and overwrite data if necessary(when you used for e.g. table segment). Or you can choose option. and use it how you want. The red button that you saw in the first photo, allows you to clear your currently made scheme and start from the beginning(use it with caution). You might want to think: what when I will do my scheme very long? I don't want to get back to the top and click on the button! Well after scrolling certain amount the "toolbar" will popup to your aid as below:





If you don't fullfil certain element pre-conditions(e.g. code tag text requirement), you will see notification in the top right corner and element won't be created until it's fixed. Be advised that notification must be closed manually. It won't disappear by itself. Some of you might find this as disadvantage but I decided to do this as notifies are not stackable(you will only see one, last notify). Personally I thought that it's better to see why something failed when I will have opportunity, rather than having in mind, that in 5 minutes, it won't be visible. Example:





Don't forget that you can switch created elements, thanks to SortableJS library! You only need to click with mouse button element and drag by holding it, until reaching desired position. By the way.. If you don't like element you've made, you can delete it by using the red cross placed on the right side of element. Be sure to pick the correct one! Demonstration of moving elements below:






You are ready now to use EzGitDoc!

Support

Browser

It is recommended to use modern browser in order to get maximum functionality out of the project. EzGitDoc uses for example classList(HTML DOM property) which is not working on every browser version. According to w3schools documentation you must have at least Chrome 8, IE10, Firefox 3.6, Safari 5.1 or Opera 11.5 in order to properly use all implemented EzGitDoc functions.

Device

EzGitDoc user interface(UI) is built using Bootstrap and MDBootstrap frameworks that allow to make the page react very nice on different screens(act responsively). If you would use EzGitDoc on smartphone it would be probably harder to work with, than on typical device with keyboard and mouse, but not impossible. The project was created with purpose to be used on for e.g. PC, notebooks. The most effective(quick and comfortable) way to use EzGitDoc will be through device with bigger screen than e.g. smartphones.

Possible questions

Can I contribute?

If you would like to take part in making EzGitDoc better, feel free to mail me what would you like to change/add (in short) or that you want to help improve EzGitDoc and we will see what we can do. I am looking forward to collaboration! (probably noone will be interested but #believe)

Can I share my template?

Yes! It's part of contribution. Email me if you have made template(or create PR but please, check first exampleTemplate and emptyTemplate files to see how to prepare it properly) that you think fits best for e.g. typical open source library or web api project. People can later see the suggestions and decide whether they want predefined, verified template or custom, they own one.
